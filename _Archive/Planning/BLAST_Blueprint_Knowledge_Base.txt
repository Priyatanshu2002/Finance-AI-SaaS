1. NORTH STAR
─────────────
Build a modular Finance AI SaaS platform that automates the most painful manual
workflows in Investment Banking, M&A, Commercial Real Estate, and Project Finance.

The platform is built as a 5-product suite with a shared infrastructure layer:

  PRODUCT 1 (Phase 1, Month 1): The "CIM Factory" (Tech-Enabled Agency)
    - **Service-First Model**: Generate institutional-grade CIMs in 48 hours for $5k flat.
    - **Tech Stack**: Claude Projects (Ingestion) + Opus 4.6 (Reasoning) + Excel Plugin (Spreading).
    - **Goal**: Immediate cash flow ($20k/mo) & "Golden Dataset" creation.
    - **Profitability Score**: 10/10 (High leverage, zero churn).

  PRODUCT 2 (Phase 2, Month 2): The "Deal Scouting" Engine (Data Service)
    - **Service Model**: Scraping & qualifying niche SMB deals for PE firms.
    - **Revenue**: Monthly retainer ($2k) + Success Fees ($50k+).
    - **Tech**: Firecrawl (Scraping) + Opus (Qualifying).

  PRODUCT 3 (Phase 3, Months 3-6): Universal Financial Data Extraction SaaS
    - **Pivot**: Built in the background using data collected from Agency work.
    - **Goal**: Convert internal agency tools into public API.
    - **Profitability Score**: 9/10 (Long-term wealth).

  PRODUCT 4 & 5 (Phase 4+): CRE & Project Finance
    - Deferred until Agency cash flow >$10k/mo.

  STRATEGIC INSIGHT: The "Agency-First" approach solves the "Trust" and "Data" problems simultaneously. We build the tech to service our own clients, then productize it once it's battle-tested.

5. BEHAVIORAL RULES (System "Do Not" Rules & Constraints)
─────────────────────────────────────────────────────────
  SECURITY:
  ├── NEVER store raw API credentials in code or database
  ├── NEVER transmit financial documents without encryption (TLS in transit, AES-256 at rest)
  ├── NEVER expose raw financial data in client-side code
  ├── ALWAYS maintain audit trails for every data access and modification
  └── ALWAYS enforce role-based access control (RBAC)

  DATA INTEGRITY:
  ├── NEVER present AI-extracted data as verified without confidence scores
  ├── ALWAYS flag low-confidence extractions for human review (<95% confidence)
  ├── NEVER auto-approve financial figures without cross-validation
  ├── ALWAYS maintain source attribution for every extracted data point
  └── ALWAYS preserve original documents immutably

  AI BEHAVIOR:
  ├── NEVER hallucinate financial figures — use only extracted/verified data
  ├── ALWAYS use deterministic tools for calculations (Python scripts, not LLM)
  ├── NEVER make investment recommendations or provide financial advice
  ├── ALWAYS include disclaimers on AI-generated content
  └── LLMs handle NLP tasks ONLY; business logic is deterministic code

  COMPLIANCE:
  ├── SOC 2 Type II compliance target
  ├── GDPR compliance for European clients
  ├── Data residency requirements per jurisdiction
  └── Regular security audits and penetration testing

  TONE & COMMUNICATION:
  ├── Professional, institutional-grade language in all reports
  ├── Technical precision in financial terminology
  ├── No casual or ambiguous language in generated documents
  └── Match industry conventions (IB-speak, CRE terminology, PF jargon)
